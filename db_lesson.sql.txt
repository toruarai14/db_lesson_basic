department_id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
name VARCHAR(20) NOT NULL,
created_at TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP ,
updated_at TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP 
);

Q6


SELECTとはレコードを表示する際に扱われ、カラムを指定することで詳細に絞り込むことができる。設問の記述に沿った場合にはテーブル内の`name`, `email`, `age`の三つの情報をレコードごとに表示されることができる。


FROMはテーブルを絞り込む際に扱われる。FROM `people`と記述されている場合は、peopleテーブルに絞り込まれている状態で、SELECTやDELETEといったほかの文と組み合わせて用いられる。


WHEREではカラムの後にその値を記述することで、その値に完全に一致するレコードを取得することができる。`department_id` = 1と記載した場合にはdepartment_idの値が1のレコードのみが表示される。


ORDER BY created_idとは指定したカラム（created_id)を基準にレコードを昇順に並び替えることができる
